I’ve been learning media queries!

Using media query I discovered the following;

  • Its good practice to design your website for your smallest screen, then use a @media break for tablets and desktops.
  • It’s important to put your @media break at the bottom, I figured this out when my screen wasn’t prioritising the values set in my @media break for screens over 900px which was positioned at the top of the page

next on the list;

  • Responsive layout for portrait and landscape phones
  • Responsive layout for tablets
  • Lazy loading

I'm currently learning about Flexbox partly because it's very important but also because I have a specific challenge in the website I'm making for an artist.

The artist would like the images to scroll vertically in two columns, and for the second column to be around 300-400 pixels further down the page, a bit like bricks.

I had no problem doing this using simple html and css however I want the artist to easily upload a new image and for it to smoothly fall into the right column, so I used nth-of-type() and floated every other image to the right and assigning the 2nd child with a top margin of 300px as demonstrated in Fig 1.

This is ok, but I run into issues when the Images are different heights so I would like to explore the layout using flexbox.
